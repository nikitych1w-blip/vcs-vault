## ADDED Requirements

### REQ-RS-001: Поиск репозиториев по строке и области проекта
**Узел**: API / web2 / Проект / Репозитории / Поиск
**Уровень**: lv6
**Теги**: #SC #api #web2 #repo #project #ПАО
**Версия API**: web2
**Компонент**: API

Система SHALL выполнять поиск репозиториев через `GET /repos/search` по параметру `q` в области доступных пользователю проектов, а при передаче `project_name` ограничивать поиск указанным проектом.

#### Scenario: Поиск по всем доступным проектам
- **WHEN** аутентифицированный пользователь вызывает `GET /repos/search` с параметром `q` и без `project_name`
- **THEN** система возвращает результаты, совпадающие с `q`, только из доступных пользователю проектов

#### Scenario: Поиск в рамках проекта
- **WHEN** аутентифицированный пользователь вызывает `GET /repos/search` с параметрами `q` и `project_name`
- **THEN** система возвращает результаты поиска только внутри указанного проекта

### REQ-RS-002: Быстрые фильтры "Избранные", "Наблюдаю", InSourceHub
**Узел**: UI / Проект / Репозитории / Поиск / Быстрые фильтры
**Уровень**: lv7
**Теги**: #SC #UI #FE #web2 #repo #project #filter
**Версия API**: web2
**Компонент**: UI

Система SHALL в web2 поддерживать быстрые фильтры списка репозиториев через параметры `is_favorite`, `is_watching`, `is_in_source_hub` в `GET /repos/search` и отображать состояние карточек по полям `is_starring`, `is_watching`, `is_in_source_hub` из ответа.

#### Scenario: Фильтр "Избранные"
- **WHEN** пользователь включает фильтр «Избранные»
- **THEN** UI отправляет `is_favorite=true` и отображает только репозитории с `is_starring=true`

#### Scenario: Фильтр "Наблюдаю"
- **WHEN** пользователь включает фильтр «Наблюдаю»
- **THEN** UI отправляет `is_watching=true` и отображает только репозитории с `is_watching=true`

#### Scenario: Фильтр InSourceHub
- **WHEN** пользователь включает фильтр `InSourceHub`
- **THEN** UI отправляет `is_in_source_hub=true` и отображает только репозитории с `is_in_source_hub=true`

### REQ-RS-003: Фильтрация по языкам и архивности
**Узел**: API / web2 / Проект / Репозитории / Поиск / Фильтрация
**Уровень**: lv7
**Теги**: #SC #api #web2 #repo #project #filter #ПАО
**Версия API**: web2
**Компонент**: API

Система SHALL поддерживать фильтрацию результатов поиска по параметрам `language` и `archived`, где `language` задаётся строкой языков через запятую без пробелов, а `archived` работает как tri-state (`true`, `false`, не передан).

#### Scenario: Фильтр по языкам
- **WHEN** пользователь вызывает `GET /repos/search` с параметром `language=typescript,python`
- **THEN** в выдачу попадают только репозитории, соответствующие переданным языкам

#### Scenario: Параметр archived не передан
- **WHEN** пользователь вызывает `GET /repos/search` без `archived`
- **THEN** система возвращает одновременно архивные и неархивные репозитории, удовлетворяющие остальным условиям

### REQ-RS-004: Сортировка и пагинация списка
**Узел**: UI / Проект / Репозитории / Поиск / Сортировка и пагинация
**Уровень**: lv7
**Теги**: #SC #UI #FE #web2 #repo #project #sorting
**Версия API**: web2
**Компонент**: UI

Система SHALL поддерживать сортировку через параметр `sort` (формат `^[a-z_]+(:asc|:desc)?(,[a-z_]+(:asc|:desc)?){0,2}$`) и постраничную навигацию через `page` и `limit`.

#### Scenario: Сценарий "Недавно обновлённые"
- **WHEN** пользователь выбирает режим «Недавно обновлённые»
- **THEN** UI отправляет `sort=update:desc` и отображает результаты по убыванию `updated_at`

#### Scenario: Невалидный sort
- **WHEN** пользовательский запрос содержит значение `sort`, не соответствующее паттерну контракта
- **THEN** система возвращает `400 BadRequest` с деталями ошибки валидации

### REQ-RS-005: Формат успешного ответа списка репозиториев
**Узел**: API / web2 / Проект / Репозитории / Поиск / Ответ
**Уровень**: lv8
**Теги**: #SC #api #web2 #repo #ПАО
**Версия API**: web2
**Компонент**: API

Система SHALL при успешном запросе возвращать `200` с телом `ReposShortList`, содержащим обязательные блоки `pagination` и `data[]`, где каждый элемент `data` соответствует контракту `RepoShortListItem`.

#### Scenario: Успешный ответ с найденными репозиториями
- **WHEN** по условиям запроса найден хотя бы один репозиторий
- **THEN** система возвращает `200` и `data[]` с обязательными полями карточки (`id`, `name`, `full_name`, `is_starring`, `is_watching`, `archived`, `updated_at`, `is_in_source_hub`)

#### Scenario: Успешный ответ без совпадений
- **WHEN** по условиям запроса совпадения не найдены
- **THEN** система возвращает `200`, пустой `data[]` и корректный объект `pagination`

### REQ-RS-006: Матрица ошибок и доступа
**Узел**: API / web2 / Проект / Репозитории / Поиск / Ошибки
**Уровень**: lv8
**Теги**: #SC #api #web2 #repo #auth #ПАО
**Версия API**: web2
**Компонент**: API

Система SHALL для `GET /repos/search` возвращать `400`, `401`, `403`, `404`, `500` по контракту UI API и использовать схему `ErrorResponse` для тела ошибки.

#### Scenario: Некорректные параметры запроса
- **WHEN** пользователь передаёт невалидные параметры (`sort` или иные query-параметры в некорректном формате)
- **THEN** система возвращает `400 BadRequest` с деталями ошибки валидации

#### Scenario: Запрос без аутентификации
- **WHEN** вызов `GET /repos/search` выполнен без валидной cookie-аутентификации web2
- **THEN** система возвращает `401 Unauthorized`

#### Scenario: Недостаточно прав на область данных
- **WHEN** пользователь аутентифицирован, но не имеет права читать запрошенные репозитории
- **THEN** система возвращает `403 Forbidden`
