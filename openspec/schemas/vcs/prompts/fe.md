## Контекст задачи

Ты создаёшь UI-артефакты OpenSpec на основе SA-требований.

При работе над каждым артефактом:

- Читай `sa/proposal.md` — источник намерения
- Читай `sa/specs/` — требования от SA, особенно узлы `#UI` и `#FE`
- Читай `be/design.md` — API-контракты, которые будет потреблять FE
- Все артефакты размещай в `changes/{change-name}/fe/`

---

## Инструкции по артефактам

### fe/design.md

```markdown
## FE Design: <название изменения>

### Компоненты
Список новых или изменяемых UI-компонентов

### Состояния
Для каждого компонента:
- loading — скелетон / спиннер
- success — основное состояние
- empty — пустой список / нет данных
- error/{код} — обработка ошибок (401, 403, 404, 500)

### API-интеграция
| Эндпоинт | Версия | Когда вызывается |
|----------|--------|-----------------|

### Данные для отображения
Какие поля из API-ответа отображаются в UI

### Навигация
Переходы между страницами, URL-параметры, query params

### Узлы test-model
Затрагиваемые узлы (только #UI и #FE):
- lv<N>: <путь> — что изменяется
```

Правила:
- Опирайся на узлы `#UI` и `#FE` из test-model
- Указывай версию API (преимущественно web2 для нового FE)
- Deprecated-узлы (`(удалить раздел)`) не реализуй
- Описывай все состояния компонента, включая ошибки
