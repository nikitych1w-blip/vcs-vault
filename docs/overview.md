# vcs-vault: обзор репозитория

## Что это такое

**vcs-vault** — Obsidian-vault команды продукта **SourceControl** (корпоративная VCS-платформа на базе Gitea).

Репозиторий выполняет три функции:

1. **База знаний** — хранит документацию о продукте, его архитектуре, API и процессах (в `vault/` и `sources/`)
2. **Конфигурация AI-ассистентов** — ролевые скиллы для Claude, которые превращают LLM в SA, QA, BE, FE-специалиста конкретно этого продукта (в `vault/skills/`)
3. **Тест-модель** — иерархический каталог всей функциональности продукта, к которому привязаны TC, требования и задачи (в `vault/test-model/`)

## Продукт: SourceControl

**SourceControl** — корпоративная платформа управления исходным кодом (VCS), построенная на базе Gitea.

### Интерфейсы

| Интерфейс | Тег | Описание |
|-----------|-----|----------|
| Web UI | `#UI` `#FE` | Браузерный интерфейс управления проектами и репозиториями |
| REST API | `#api` | Версии: v1, v2, v3, web1, web2 (новый FE) |
| CLI | `#CLI` | Командный интерфейс (git) |

### Приоритетные (`#ПАО`) поверхности

- REST API v1 и v2
- Frontend web2 (новый платформенный FE)
- Git API
- Kafka API

### Ключевые интеграции

| Система | Тег |
|---------|-----|

| Kafka | `#kafka` `#ПАО` |
| Jenkins / Pipeliner | `#pipeline` |
| SonarQube | `#sonar` |

## Структура директорий

```
vcs-vault/
├── docs/                     # Документация по самому vault (этот раздел)
├── vault/                    # Основная база знаний (Obsidian vault)
│   ├── skills/               # Ролевые скиллы для Claude (common, sa, be, fe, qa)
│   ├── knowledges/           # Глубокие технические знания по подсистемам (fe, onbording)
│   └── test-model/           # Иерархическая тест-модель продукта
├── sources/                  # Внешние источники знаний (клонируются через vault-cli)
├── openspec/                 # OpenSpec: схема артефактов и изменения
│   ├── config.yaml           # Генерируется из vault/skills/ (не редактировать вручную)
│   ├── schemas/vcs/          # Схема артефактов для схемы vcs
│   └── changes/              # История изменений (каждое — своя директория)
├── vault-cli/                # Go-бинарь sync для управления источниками
├── sources.yaml              # Реестр всех внешних источников
├── Makefile                  # Все команды управления
└── vault.canvas              # Карта vault (Obsidian Canvas)
```

## Как открыть

1. Клонировать репозиторий
2. Загрузить внешние источники: `make clone`
3. Открыть папку как vault в **Obsidian**

Зависимости для полного функционала:
- Go 1.22+ (vault-cli)
- git
- pandoc (wiki-адаптер: PDF → Markdown)
- SSH-ключ для внутренних репозиториев Sberbank (`vcs-*`)
- `SBERTRACK_TOKEN` (wiki-источники)
- Node.js + `openspec` CLI (`~/.npm-global/bin/openspec`)

Obsidian-конфиг (`vault/.obsidian/`) находится в `.gitignore` — каждый участник настраивает его локально.
