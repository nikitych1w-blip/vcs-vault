# Тест-модель продукта

## Что такое тест-модель

**Тест-модель** — иерархический каталог всей функциональности продукта SourceControl. Это единая система координат для всей команды: SA, QA, BE, FE.

Живёт в `vault/test-model/`. Структура директорий = иерархия узлов.

Каждый артефакт в проекте (требование, тест-кейс, дефект, задача разработки) обязан содержать координату: путь узла + уровень (lv) + теги.

---

## Иерархия уровней

| Уровень | Что описывает | Примеры |
|---------|---------------|---------|
| **lv1** | Интерфейс / подсистема | UI, API, CLI, Общие механизмы, Ролевая модель |
| **lv2** | Крупный раздел / версия | Проект, Dashboard, v1/v2/v3/web1/web2, Kafka, Git |
| **lv3** | Секция внутри раздела | Репозитории, Настройки, Группы привилегий |
| **lv4** | Конкретная сущность | Репозиторий (отдельный), Webhook, Аватар |
| **lv5** | Область функциональности | Код, Запросы на слияние, Сравнение, Активность |
| **lv6** | Фича | Ветки, Коммиты, Теги, Лицензии, Pipeline status |
| **lv7** | Конкретная операция | Создание ветки, Удаление тега, Откат коммита |
| **lv8** | Атомарный сценарий / ограничение | Список, Фильтры, Ограничение длины, Символы |

Пример пути: `UI / Проект / Репозитории / Репозиторий / Код / Ветки / Название / По символам` → lv8

## Frontmatter узла

Каждый файл узла содержит frontmatter:

```yaml
---
tags:
  - SC          # принадлежность продукту
  - UI          # интерфейс
  - branch      # сущность
  - ПАО         # приоритет
lv: 7
status: covered   # covered / partial / missing
---
```

Статусы покрытия:
| Статус | Значение |
|--------|----------|
| `covered` | Полное покрытие тест-кейсами |
| `partial` | Покрыт частично |
| `missing` | Тест-кейсов нет |

**Узел с `#ПАО` + `status: missing` блокирует релиз.**

## Использование в артефактах

### В требованиях SA (sa-specs)

```markdown
### REQ-001: Поиск репозиториев по имени
**Узел**: API / web2 / Repos / Search
**Уровень**: lv5
**Теги**: #api #web2 #ПАО #repo
```

### В тест-кейсах QA

```markdown
---
id: TC-1234
node: UI / Проект / Репозитории / Репозиторий / Код / Ветки / Создание
lv: 7
tags: [UI, SC, branch, ПАО, web2]
api_version: web2
status: covered
---
```

### В задачах BE/FE

```markdown
## be-tasks
- [ ] 1.1 Роутинг: зарегистрировать GET /repos/search
  - Узел: API / web2 / Repos / Search (lv5)
```

## Глубина работы по ролям

| Роль | Уровни |
|------|--------|
| SA — декомпозиция, границы модуля | lv1–lv3 |
| SA — фичи, API-контракты, use cases | lv4–lv6 |
| **SA — алгоритмы, валидации, бизнес-правила** | **lv6–lv8** |
| QA — smoke-тесты, чек-листы | lv3–lv5 |
| **QA — основной уровень TC** | **lv7–lv8** |

---

## Deprecated узлы

Узлы с пометкой `(удалить раздел)` — deprecated:
- SA не анализирует
- QA не пишет TC, деактивирует существующие
- BE/FE не реализуют новую функциональность
