# Инструкция по использованию промта для автоматизации тестирования API

## Обзор

Данный промт (`promt_v3.txt`) предназначен для автоматизации полного цикла тестирования API через систему управления тестированием (TMS) с использованием агента Cline. Промт реализует 5-этапный процесс с обязательной валидацией на каждом шаге.

## Структура проекта

```
api_agent_vcs_v3/
├── promt_v3.txt              # Основной промт с инструкциями
├── shablon.md                # Шаблон для тест-кейсов
├── test_results_shablon.json # Шаблон отчёта о тестировании
├── config.md                 # Конфигурация (учётные данные)
├── generate_vcs_data.py      # Скрипт генерации тестовых данных
├── import_tests_to_tms_full.py # Скрипт импорта в TMS
├── openapi_http_v3_bundle.yaml # OpenAPI спецификация HTTP v3
└── openapi_ui_bundle.yaml    # OpenAPI спецификация UI v2
```

## Требования

- Python 3.x (для скриптов импорта и генерации данных)
- Доступ к TMS: `https://portal.works.prod.sbt`
- Доступ к VCS API: `https://sc-vp.ift.pd04.pvw.sbt`
- Учётные данные в файле `config.md`

## Этапы выполнения

### Этап 1: Генерация тест-кейсов

**Входные данные:**
- OpenAPI спецификация (`openapi_http_v3_bundle.yaml` или `openapi_ui_bundle.yaml`)
- HTTP метод и эндпоинт (например, `POST /members`)

**Техники тест-дизайна:**
- ✅ Эквивалентное разделение
- ✅ Граничные значения
- ✅ Попарное тестирование
- ✅ Предугадывание ошибок
- ✅ Тестирование на основе состояний
- ❌ Исключены: проверки на инъекции

**Результат:** Файл `[METHOD]_[ENDPOINT].md` с тест-кейсами по шаблону `shablon.md`

**Валидация:** Агент запросит подтверждение после генерации файла.

### Этап 2: Анализ структуры папок в TMS

**Действия агента:**
1. Выполняет POST запрос к API TMS для получения структуры папок
2. Определяет путь для новой папки:
   - Для `openapi_ui_bundle.yaml`: `SC v2 → 2. api web → 4. v2 → [новая папка]`
   - Для `openapi_http_v3_bundle.yaml`: `C v2 → 1. API → 2. api v3 → [новая папка]`
3. Проверяет существование папки

**Валидация:** Агент покажет путь и ID папки для подтверждения.

### Этап 3: Импорт тест-кейсов в TMS

**Параметры скрипта `import_tests_to_tms_full.py`:**

| Аргумент | Описание |
|----------|----------|
| `--create-cycle` | Флаг создания цикла тестов |
| `--username` | Логин пользователя (из config.md) |
| `--password` | Пароль пользователя (из config.md) |
| `--folder` | UUID папки (если существует) |
| `--folder-name` | Название папки: "[METHOD] [endpoint]" |
| `--parent-folder` | ID родительской папки (если папка не существует) |
| `--cycle-name` | Название цикла тестов |
| `[путь к файлу]` | Путь к файлу с тест-кейсами |

**Предварительная валидация:**
Агент покажет:
- Путь до папки с тест-кейсами
- Таблицу тест-кейсов (Test ID, Название, Приоритет, Техника)
- Параметры импорта

**После импорта:** Агент обновит файл с тест-кейсами, заменив временные ID на полученные из TMS.

**Валидация:** Два запроса подтверждения - перед запуском и после обновления ID.

### Этап 4: Генерация тестовых данных VCS

**Параметры скрипта `generate_vcs_data.py`:**

```bash
python generate_vcs_data.py \
  --base-url "https://sc-vp.ift.pd04.pvw.sbt/api/v3" \
  --tenant "2afda79a-ae63-47f8-b0d3-1f67ea07b92c" \
  --project "agentvp"
```

**Генерируемые данные:**
- 1 репозиторий
- 50 веток в репозитории
- 3 файла в каждой ветке (всего 150 файлов)
- 25 Pull Request'ов

**Валидация:** Агент покажет сводку созданных данных.

### Этап 5: Прогон тестов и формирование отчёта

**Параметры выполнения:**
- **Base URL:** `https://sc-vp.ift.pd04.pvw.sbt/web/v2/`
- **Authorization:** Bearer токен (должен быть актуальным)

**Требования:**
- ✅ Использовать только данные из Этапа 4
- ✅ Выполнить ВСЕ тест-кейсы из Этапа 1
- ✅ Не изменять ожидаемые результаты
- ✅ Фиксировать фактические результаты

**Результат:** Файл отчёта по шаблону `test_results_shablon.json`:

```json
{
  "summary": {
    "total": 10,
    "passed": 8,
    "failed": 2,
    "success_rate": "80.00%"
  },
  "results": [
    {
      "test_id": "TC-001",
      "name": "Название теста",
      "passed": true,
      "expected_status": 200,
      "actual_status": 200,
      "response": "{\"key\": \"value\"}"
    }
  ]
}
```

## Критические правила

1. 🛑 **НИКОГДА** не переходить к следующему этапу без явного подтверждения пользователя
2. 🛑 **НИКОГДА** не изменять ожидаемые результаты тестов из Этапа 1
3. 🛑 **ВСЕГДА** использовать актуальный Bearer токен для Этапа 5
4. 🛑 **ВСЕГДА** использовать только сгенерированные данные Этапа 4
5. ✅ **ВСЕГДА** запрашивать валидацию в явном виде после каждого этапа

## Устранение неполадок

### Ошибка авторизации в TMS
- Проверьте учётные данные в `config.md`
- Убедитесь, что срок действия пароля не истёк

### Ошибка Bearer токена (Этап 5)
- Запросите новый токен через API аутентификации
- Обновите токен в параметрах выполнения

### Скрипт импорта не найден
- Убедитесь, что `import_tests_to_tms_full.py` находится в рабочей директории
- Проверьте права на выполнение скрипта

### Ошибка генерации данных VCS
- Проверьте доступность API: `https://sc-vp.ift.pd04.pvw.sbt/api/v3`
- Убедитесь, что tenant и project указаны корректно
