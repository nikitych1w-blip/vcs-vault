# Гайд для аналитиков

Этот репозиторий (`vcs-api`) --- единственный источник правды для OpenAPI-спецификаций SourceControl.
Вы описываете ручки здесь, открываете PR, а после мержа спека автоматически попадает в `gitea` и `vcs-playwright` через submodule.

## Структура спецификаций

```
api/openapi/
├── http/
│   ├── v2/                          # API v2
│   │   ├── index.yaml               # корневой файл спеки
│   │   ├── domains/                  # домены (branches, pulls, repos, ...)
│   │   │   └── <domain>/
│   │   │       ├── routes.yaml       # пути и операции
│   │   │       ├── schemas.yaml      # модели данных
│   │   │       └── responses.yaml    # ответы
│   │   └── common/                   # общие параметры, ошибки, ...
│   └── v3/                          # API v3 (аналогичная структура)
├── ui/                              # UI API (аналогичная структура)
└── ...
```

Вы работаете с файлами в `domains/` нужной поверхности (`v2`, `v3` или `ui`).
Корневой `index.yaml` подключает домены через `$ref`.

## Черновые ручки: `x-internal: true`

Если ручка описана в спеке, но ещё не готова к реализации на BE, помечайте её флагом `x-internal: true`.
Этот флаг исключает операцию из BE codegen. Снимается, когда BE берёт ручку в работу.

### Правила

- Флаг ставится **на уровне конкретной операции** (`get`, `post`, `put`, `patch`, `delete`), а не на уровне path.
- Если на одном path часть методов черновые, а часть рабочие --- помечаем только черновые.
- Если весь path черновой --- всё равно ставим `x-internal: true` у **каждого метода отдельно**.
- Теги (`tags`) не трогаем --- оставляем как есть.


## Make-команды

### Сборка (lint + bundle)

```bash
make build
```

Запускает линтер (`redocly lint`) и собирает bundle-файлы из исходных YAML.

### Проверка обратной совместимости


Сравнивает текущие изменения с `origin/master` и проверяет, что вы не сломали обратную совместимость (удалённые поля, изменённые типы, убранные эндпоинты и т.д.).

Можно проверить конкретную поверхность:

```bash
make check-api-compat-v2
make check-api-compat-v3
make check-api-compat-ui

-------------------------
make check-api-compat-all
```

### Генерация Postman-коллекций

Собирает спеки и генерирует Postman-коллекции в `postman/generated/`.

```bash
make postman
```

### Всё вместе

```bash
make all
```

Выполняет `build` + `check-api-compat-all` + `postman` за одну команду.

## Чеклист перед PR

1. `make build` --- линтер и бандл проходят без ошибок.
2. `make check-api-compat-all` --- обратная совместимость не нарушена (или нарушение согласовано).
3. Черновые ручки помечены `x-internal: true` на уровне каждой операции.
4. Теги не менялись без согласования.
5. PR описывает, какие ручки добавлены/изменены.
