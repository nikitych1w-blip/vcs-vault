# BaseComponent

## Обзор

`Element` — это обертка над Playwright `Locator`, которая предоставляет улучшенный интерфейс для работы с элементами UI. Она используется как базовый класс для создания компонентов пользовательского интерфейса в тестах.

## Цель создания

Основная цель создания `Element` - улучшение отчетности в Allure. Благодаря использованию `step` из `@vcs-pw/test`, все действия над элементами становятся более информативными в отчетах, что значительно упрощает диагностику проблем при падении тестов.

Кроме того, `Element` предоставляет:

- Единый интерфейс для работы с элементами UI
- Удобную иерархию компонентов
- Автоматическое формирование информативных имен для элементов
- Расширенный функционал по сравнению с обычным Locator

## Особенности

### 1. Улучшенная отчетность в Allure

Все методы `BaseComponent` оборачиваются в `step`, что позволяет видеть в отчетах более подробную информацию о том, что именно происходит во время выполнения теста. Например:

```typescript
// Вместо простого вызова
await element.click();

// В отчете будет видно:
// "Клик по элементу 'Кнопка отправки'"
```

### 2. Именование элементов

Каждый элемент имеет имя (`name`), которое используется для создания информативных шагов в отчетах. При создании дочерних элементов имя автоматически формируется как комбинация родительского имени и суффикса.

### 3. Расширенный функционал

`BaseComponent` предоставляет расширенный набор методов для работы с элементами, включая:

- Методы для взаимодействия с элементами (click, fill, hover и т.д.)
- Методы для проверки состояний элементов (isVisible, isDisabled, isChecked и т.д.)
- Методы для получения информации о элементах (boundingBox, allInnerTexts и т.д.)
- Методы ожидания (expect и softExpect)

### 4. Поддержка цепочек вызовов

Благодаря методам `first()`, `last()`, `nth()`, `and()`, `or()`, `filter()` можно создавать сложные селекторы и работать с коллекциями элементов.

## Использование

### Тип ParentType

Для создания компонентов используется тип `ParentType`, который может быть:

- `Page` - страница Playwright
- `Locator` - локатор Playwright
- `FrameLocator` - локатор фрейма Playwright

Это позволяет создавать компоненты, которые могут быть инициализированы как на уровне страницы, так и на уровне других компонентов.

### Создание базового компонента

```typescript
import Element from '@vcs-pw/ui/components/base.component';

class MyComponent extends Element {
  constructor(base: ParentType) {
    super({
      name: 'Мой компонент',
      locator: base.getByTestId('ComponentTestId'),
    });
  }
}
```

### Создание дочернего компонента

В компонентах используется паттерн, где дочерние элементы создаются относительно базового локатора родителя. Это позволяет создавать иерархию компонентов, где каждый элемент знает о своем месте в структуре.

```typescript
// Пример из Navbar component
constructor(base: ParentType) {
  super({
    name: 'Панель навигации',
    locator: base.locator('#navbar'),
  });

  // Дочерние элементы создаются относительно локатора родителя (navbar)
  this.notifications = this.child('Уведомления', (base) => base.getByRole('link', { name: 'Уведомления' }));
  this.settings = this.child('Профиль и настройки', (base) =>
    base.getByRole('menu', { name: 'Профиль и настройки...' }),
  );
}
```

### Метод child

Метод `child` - это protected метод базового класса `Element`, который используется для создания дочерних элементов. Он принимает два параметра:

1. `nameSuffix` - суффикс для имени дочернего элемента
2. `locatorOrFactory` - либо готовый `Locator`, либо функция, которая принимает базовый `Locator` и возвращает новый `Locator`

Пример использования метода `child`:

```typescript
// Создание дочернего элемента с использованием функции-фабрики
this.button = this.child('Кнопка', (base) => base.locator('[data-testid="submit-button"]'));

// Создание дочернего элемента с готовым локатором
this.input = this.child('Поле ввода', someReadyLocator);
```

Этот подход позволяет создавать сложные иерархии компонентов, где каждый дочерний элемент наследует имя родителя и добавляет свою специфику.

## Принцип работы

### Поиск элементов относительно базового локатора

В отличие от традиционного подхода, где передаются готовые локаторы, компоненты в этом проекте используют паттерн, при котором дочерние элементы создаются относительно базового локатора родителя. Это позволяет:

1. **Упростить создание компонентов**: достаточно передать базовый элемент (Page, Locator или FrameLocator)
2. **Создать иерархию**: каждый дочерний элемент знает о своем месте в структуре
3. **Улучшить отчетность**: имена элементов формируются автоматически с учетом иерархии

### Пример создания компонента с дочерними элементами

```typescript
import { ParentType } from '@vcs-pw/ui';
import Element from '@vcs-pw/ui/components/base.component';

class MyComponent extends Element {
  readonly button: Element;
  readonly input: Element;

  constructor(base: ParentType) {
    super({
      name: 'Мой компонент',
      locator: base.getByTestId('my-component'),
    });

    // Дочерние элементы создаются относительно базового локатора
    this.button = this.child('Подтвердить', (base) => base.getByTestId('submit-button'));
    this.input = this.child('Поле ввода', '#input-field');
  }
}
```

В этом примере:

- `this.button` будет иметь имя "Мой компонент > Кнопка"
- `this.input` будет иметь имя "Мой компонент > Поле ввода"

Это обеспечивает понятную иерархию в отчетах Allure.
