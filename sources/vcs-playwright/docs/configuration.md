# Конфигурация

Этот документ описывает систему загрузки и обработки конфигурации в приложении.

Поддерживается:

- Многоуровневая загрузка из YAML
- Динамические placeholder'ы (`{{ config:... }}`, `{{ vault:... }}` и др.)
- Переопределение через переменные окружения
- Кэширование запросов к Vault
- Валидация по схеме

## Структура загрузки

Конфигурация формируется поэтапно:

1. Загрузка YAML-файлов (по порядку, с глубоким слиянием. env `VCS_CONFIG` — путь к конфигам, через запятую)
2. Разрешение placeholder'ов:
   1. `{{ config:... }}` — значение из другого поля конфига
   2. `{{ vault:... }}` — секрет из HashiCorp Vault
   3. `{{ env:... }}` — переменная окружения
   4. `{{ file:... }}` — содержимое файла
3. Переопределение через `VCS__SECTION__KEY=value`. Префикс определяется переменной окружения `CONFIG_ENV_PREFIX`. По умолчанию `VCS`.
4. Валидация по схеме Zod

## Поддерживаемые источники

| Протокол | Пример                      | Описание                        |
| -------- | --------------------------- | ------------------------------- |
| config:  | {{ config:db.host }}        | Ссылка на другое поле в конфиге |
| vault:   | {{ vault:db:password }}     | Секрет из Vault (путь:ключ)     |
| env:     | {{ env:PORT }}              | Переменная окружения            |
| file:    | {{ file:/certs/token.pem }} | Содержимое файла                |

## Vault: кэширование запросов

Чтобы минимизировать количество запросов к HashiCorp Vault, используется внутреннее кэширование:

```yaml
database:
  user: '{{ vault:secret_path:username }}'
  pass: '{{ vault:secret_path:password }}'
  host: '{{ vault:secret_path:host }}'
```

Будет выполнен только один запрос к secret_path, остальные значения берутся из кэша.

## Переменные окружения

Можно переопределять поля через переменные окружения с префиксом VCS\_\_.

Формат
VCS\_\_SECTION\_\_KEY=value
→ Преобразуется в section.key

Пример

```env
VCS__DB__HOST=prod-db.example.com
VCS__APP__PORT=8080
```

После загрузки

```ts
{
  db: { host: 'prod-db.example.com' },
  app: { port: '8080' }
}
```

## Глубокое слияние файлов

Несколько конфигов можно передать — они будут объединены глубоко:

```ts
await loadConfig(vaultService, 'config/base.yaml', 'config/prod.yaml');
```

prod.yaml имеет приоритет над base.yaml.

Вложенные объекты не перезаписываются полностью — объединяются.

## Валидация

Конфиг обязательно проходит валидацию по схеме ConfigSchema (на основе zod).
