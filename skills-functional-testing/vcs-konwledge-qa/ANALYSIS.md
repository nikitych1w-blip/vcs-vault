## Ключевые выводы

### Для QA-агента:

1. **Паттерны шагов (обязательно использовать):**
   - Нажатие кнопки: "Нажать [на кнопку/элемент]"
   - Навигация: "Перейти на [страницу]"
   - Заполнение формы: "Заполнить [поле]"
   - Авторизация: "Авторизоваться под [ролью]"

2. **Паттерны результатов (обязательно использовать):**
   - Отображение: "=> Отображается [элемент]"
   - Успешная операция: "=> [Элемент] успешно [действие]"
   - Уведомление: "=> [Текст уведомления]"
   - Валидация: "=> Выводится [ошибка / предупреждение]"

3. **Структура тест-кейсов:**
   - Большинство TC (47%) не имеют шагов - это заглушки/черновики
   - Типичный TC имеет 3-7 шагов
   - 85% TC имеют приоритет Major/Minor/Critical

4. **Типы тестов:**
   - UI: 34% из всех с указанным уровнем
   - API: 5% из всех с указанным уровнем

5. **Статусы:**
   - 23.7% TC имеют статус "Актуальный" (для тестирования)
   - 48.3% имеют пустой статус (неизвестно)
   - 8.2% - "Неактуальный" (не использовать)

## Рекомендации

1. **Для генерации тест-кейсов:**
   - Использовать паттерны из `patterns.md`
   - Указывать версию UI (web1 или web2)
   - Добавлять тег `agent-generated`

2. **Для импорта в TMS:**
   - Использовать путь: `SC v2 → 1. UI → [название] → [версия]`
   - Учитывать существующие папки

3. **Для выполнения тестов:**
   - Использовать только тест-кейсы со статусом "Актуальный"
   - Проверять наличие заглушек (0 шагов)
   - Делать скриншоты при дефектах
