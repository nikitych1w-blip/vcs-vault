# Терминология SourceControl

Глоссарий составлен на основе тестовой модели (`docs/obsidian/test-model/0. SourceControl/`).

---

## Продукт

| Термин | Тег | Описание |
|--------|-----|----------|
| **SourceControl** | `#SC` | Система управления исходным кодом — основной продукт |

---

## Интерфейсы доступа

| Термин | Тег | Описание |
|--------|-----|----------|
| **UI** | `#UI` | Веб-интерфейс пользователя |
| **API** | `#api` | Программный интерфейс; включает версии v1, v2, v3, web1, web2 (Новый FE), а также интеграции git и kafka |
| **CLI** | `#cli` | Интерфейс командной строки |

---

## Сущности верхнего уровня

| Термин          | Тег        | Описание                                                                          |
| --------------- | ---------- | --------------------------------------------------------------------------------- |
| **Проект**      | `#project` | Организационная единица, объединяющая репозитории, участников и настройки доступа |
| **Репозиторий** | `#repo`    | Хранилище исходного кода на основе Git, входящее в состав проекта                 |

---

## Объекты репозитория

| Термин                | Тег                 | Описание                                                                                    |
| --------------------- | ------------------- | ------------------------------------------------------------------------------------------- |
| **Ветка**             | `#branch`           | Именованная ссылка на последовательность коммитов; используется для параллельной разработки |
| **Коммит**            | `#commit`           | Снимок состояния кода с метаданными: автор, время, сообщение                                |
| **Тег**               | `#tags`             | Именованная метка, фиксирующая конкретный коммит                                            |
| **Релиз**             | `#realese`          | Версия продукта, связанная с тегом; может содержать артефакты и описание изменений          |
| **Запрос на слияние** | `#merge` `#request` | Запрос на слияние одной ветки с другой (аналог Pull Request / Merge Request)                |
| **Метки**             | `#marker`           | Пользовательские метки-ярлыки для категоризации запросов на слияние и этапов                |
| **Этапы**             | —                   | Вехи (milestones), группирующие запросы на слияние по циклам разработки                     |
| **Активность**        | —                   | Журнал событий репозитория (коммиты, ветки, теги, MR и др.)                                 |
| **Форк**              | `#fork`             | Копия репозитория в рамках другого проекта для независимой разработки                       |

---

## Разделы раздела «Код» репозитория

| Термин | Описание |
|--------|----------|
| **Структура** | Файловое дерево репозитория для текущей ветки/коммита |
| **Ветки** | Список веток репозитория с возможностью создания и удаления |
| **Коммиты** | История коммитов репозитория |
| **Теги** | Список тегов репозитория |
| **Релизы** | Список релизов, привязанных к тегам |
| **Метки** | Раздел управления ярлыками для MR |
| **Лицензии** | Раздел управления лицензиями репозитория |
| **Языки программирования** | Отображение доли используемых языков в репозитории (определяется плагином) |

---

## Действия с файлами (через UI)

| Термин | Описание |
|--------|----------|
| **Создание первого файла** | Создание файла в пустом репозитории через веб-интерфейс |
| **Редактирование файлов** | Редактирование существующего файла непосредственно в UI |
| **Push файлов** | Загрузка файлов в репозиторий через UI |

---

## Скачивание репозитория

| Термин | Описание |
|--------|----------|
| **HTTPS** | Клонирование репозитория по протоколу HTTPS |
| **SSH** | Клонирование репозитория по протоколу SSH |
| **Архивы** | Скачивание снимка репозитория в виде zip- или tar-архива |

---

## Флаги репозитория

| Термин | Тег | Описание |
|--------|-----|----------|
| **Избранное** | `#favorite` | Добавление репозитория в список избранных |
| **Следить** | `#follow` | Подписка на уведомления о событиях в репозитории |
| **Форкнуть** | `#fork` | Создание форка репозитория в другой проект |

---

## Ветки — свойства и ограничения

| Термин | Описание |
|--------|----------|
| **Название** | Правила именования ветки при создании |
| **Ограничение по длине** | Максимально допустимая длина имени ветки |
| **По символам** | Ограничение на допустимые символы в имени ветки |
| **По окончанию** | Ограничение на суффикс (окончание) имени ветки |
| **Ветка по умолчанию** | Основная ветка репозитория (например, `main` / `master`) |
| **Защита веток** | Правила, ограничивающие прямые push и force-push в защищённую ветку |
| **PR upstreame** | Pull Request из форка в upstream-репозиторий |

---

## Коммиты — действия

| Термин | Описание |
|--------|----------|
| **Откат** | Revert коммита — создание нового коммита, отменяющего изменения |
| **Перенос** | Cherry-pick — перенос изменений коммита на другую ветку |
| **Создать ветку** | Создание новой ветки от выбранного коммита |
| **Создать тег** | Создание тега из выбранного коммита |

---

## Теги — действия и свойства

| Термин | Описание |
|--------|----------|
| **Добавить тег** | Создание нового тега через UI-форму |
| **Создать тег** | Создание тега из коммита (в контексте сущности коммита) |
| **Удалить тег** | Удаление существующего тега |
| **Счетчик** | Отображение общего числа тегов в репозитории |

---

## Запросы на слияние — состояния и действия

| Термин | Описание |
|--------|----------|
| **Новый запрос на слияние** | Форма создания нового MR с выбором исходной и целевой ветки |
| **Переход на сравнение** | Переход к экрану сравнения двух веток перед созданием MR |
| **Открытые** | Запросы на слияние в статусе «открыт» |
| **Закрытые** | Запросы на слияние в статусе «закрыт» (merged или отклонённые) |

---

## CI/CD интеграция

| Термин | Описание |
|--------|----------|
| **Pipeline status** | Статус последнего пайплайна, связанного с веткой или коммитом |
| **Сборка** | Процесс сборки в рамках CI/CD пайплайна |
| **Sonar** | Раздел отображения результатов статического анализа кода |
| **Метрики** | Показатели качества кода, получаемые из SonarQube |
| **Подключение к SonarQube** | Настройка интеграции репозитория с SonarQube |
| **Интеграция с Jenkins и Pipeliner** | Настройка вебхуков для автоматического запуска Jenkins/Pipeliner |

## Настройки репозитория

| Термин | Описание |
|--------|----------|
| **Основные параметры** | Базовые настройки репозитория: имя, описание |
| **Расширенные** | Дополнительные настройки репозитория |
| **Видимость** | Уровень доступности репозитория (публичный / приватный / внутренний) |
| **Настройки подписи** | Параметры GPG-подписи коммитов |
| **Code Owners** | Файл/правила, определяющие владельцев кода для обязательного ревью |
| **Настройки администратора** | Административные параметры, доступные только администраторам |
| **Upstream** | Вышестоящий репозиторий, с которым связан данный форк |
| **Ключи развёртывания** | SSH-ключи с ограниченным доступом, используемые при деплое |
| **Ревью** | Настройки процесса код-ревью: обязательные ревьюеры, политики |
| **Webhooks** | Настройка HTTP-уведомлений о событиях репозитория |
| **Удаление** | Удаление репозитория |

---

## Настройки проекта
