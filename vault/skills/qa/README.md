# Ручное функциональное тестирование

Эта папка содержит скиллы для ручного функционального тестирования в рамках команды SourceControl.

## Структура

- **BASE-TESTING.md** — базовый скилл, общий для всех типов тестирования
- **API-SWAGGER-TESTING.md** — скилл для тестирования API по OpenAPI/Swagger спецификации
- **UI-FRONTEND-TESTING.md** — скилл для тестирования фронтальных сценариев

## Как использовать

Каждый скилл содержит:
- **Роль и зоны ответственности** — что должен делать тестировщик
- **Жизненный цикл тестирования** — пошаговая инструкция
- **Форматы артефактов** — шаблоны для TC, баг-репортов, чек-листов
- **Принципы работы** — ключевые правила
- **Чего избегать** — распространённые ошибки

## Интеграция с test-model

Все скиллы ориентированы на работу с тестовой моделью SourceControl:
- Координата узла обязательна для всех артефактов
- Уровни lv7–lv8 — основной фокус для функциональных тест-кейсов
- Связь: узел test-model → тест-кейс → дефект

## Связанные источники

- `vault/test-model/` — тестовая модель продукта
- `vault/skills/common/model.md` — система координат (узлы/lv/теги)
- `faq/` — справочные материалы и сценарии
- `sources/vcs-api/` — OpenAPI-спеки (для API-тестирования)
