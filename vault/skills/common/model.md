# Система координат

Единая система отсчёта для всей команды. Каждый артефакт — требование, тест-кейс, дефект, задача разработки — обязан иметь координату в этой модели.

Модель живёт в `knowledges/test-model/`. Структура директорий — это и есть иерархия узлов.

---

## Иерархия уровней

| Уровень | Что описывает | Примеры |
|---------|---------------|---------|
| **lv1** | Интерфейс / подсистема | UI, API, CLI, Общие механизмы, Ролевая модель |
| **lv2** | Крупный раздел / версия API | Проект, Dashboard, Обзор, v1/v2/v3/web1/web2, Kafka, Git |
| **lv3** | Секция внутри раздела | Репозитории, Настройки, Группы привилегий |
| **lv4** | Конкретная сущность | Репозиторий (отдельный), Webhook, Аватар |
| **lv5** | Область функциональности | Код, Запросы на слияние, Сравнение, Активность, Создание, Удаление |
| **lv6** | Фича | Ветки, Коммиты, Теги, Лицензии, Pipeline status, Метки, Структура |
| **lv7** | Конкретная операция | Создание ветки, Удаление тега, Откат коммита, Открытые MR |
| **lv8** | Атомарный сценарий / ограничение | Список, Фильтры, Ограничение длины имени, Символы в названии |

**Как читать путь узла:** `Интерфейс / Раздел / ... / Операция / Сценарий`
Пример: `UI / Проект / Репозитории / Репозиторий / Код / Ветки / Название / По символам` → lv8

## Теги

Каждый узел несёт теги — они же используются в требованиях, TC, дефектах, задачах.

### Интерфейс
| Тег | Значение |
|-----|----------|
| `#UI` | Веб-интерфейс |
| `#api` | REST API |
| `#CLI` | Командная строка |
| `#FE` | Frontend-специфика |

### Версии API
| Тег | Значение |
|-----|----------|
| `#v1` | REST API v1 |
| `#v2` | REST API v2 |
| `#v3` | REST API v3 |
| `#web1` | Frontend API web1 |
| `#web2` | Frontend API web2 (Новый FE) |
| `#git` | Git-интеграция |
| `#kafka` | Kafka-интеграция |

### Сущности
`#project` `#repo` `#branch` `#commit` `#tags` `#merge` `#request` `#fork` `#marker` `#pipeline` `#sonar` `#license` `#auth` `#ldap` `#jwt` `#access-token`

### Специальные
| Тег | Значение |
|-----|----------|
| `#SC` | Принадлежность продукту SourceControl |
| `#ПАО` | Приоритетное покрытие / проверка обязательна |

---

## Статусы покрытия узла

Проставляются в frontmatter файла узла:

| Статус | Значение |
|--------|----------|
| `status: covered` | Полное покрытие тест-кейсами |
| `status: partial` | Покрыт частично |
| `status: missing` | Тест-кейсов нет |

Узел с `#ПАО` и `status: missing` → **блокирует релиз**.

## Уровни тестирования по глубине

| Диапазон | Применение |
|----------|------------|
| lv1–lv2 | Общий контекст, вне тест-кейсов |
| lv3–lv5 | Smoke-тесты, чек-листы |
| lv6–lv7 | Функциональные тест-кейсы |
| **lv7–lv8** | **Основной уровень TC** |
