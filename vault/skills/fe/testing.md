# FE · Тесты

В репозитории для FE присутствует **Playwright** (`@playwright/test`, конфиг `playwright.config.js`) и `jsdom`.

## Что покрывать (корнер-кейсы)

Для новой страницы/фичи:
- позитивный сценарий (данные отображаются);
- состояния `empty` / `error` / `no-access` (см. [[data-state]]);
- ошибки API: 400 (валидация формы), 401, 403, 404, 409, 500 — корректная реакция UI;
- фильтры/пагинация/сортировка, если есть.

## Playwright

- E2E / интеграционные сценарии страницы. Конфиг — `playwright.config.js` в корне репо.
- Моки ответов API — каталог `mocks/__files/` (напр. `mocks/__files/repos-search/`), переиспользовать имеющиеся фикстуры.
- Трассируемость: связывать сценарий с узлом test-model и TC (как у QAA), если применимо.

## ⚠️ Уточнить (нет однозначного сигнала в репо)

- **Юнит-раннер компонентов** (vitest/jest) в зависимостях **не обнаружен** (есть `jsdom`, но не сам раннер). Прежде чем писать юнит-тесты компонентов — **уточнить у разработчика**, чем они гоняются (или ограничиться Playwright component/e2e). Не выдумывать раннер.

## Перед PR

- Прогнать тесты Playwright и линты ([[review]]); тесты должны проходить в CI.
