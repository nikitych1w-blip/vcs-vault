# FE · Данные и состояния (React Query)

Данные тянутся через **TanStack React Query** сгенерированными Orval-хуками (см. [[api-client]]). Каждый экран обязан обрабатывать все состояния, не только happy path.

## React Query

- **Чтение**: сгенерированный `useGet…Query` (из `shared/api/generated/methods`).
- **Запись**: `use…Mutation` + инвалидация связанных query (`queryClient.invalidateQueries`).
- Кэш/ключи — из сгенерированных хуков; не дублировать запросы руками.
- Ошибки HTTP приходят через `customInstance` (axios) — единый формат (см. `shared/api/axiosClient.ts`).

## Обязательные состояния экрана

| Состояние | Условие | Что показать |
|-----------|---------|--------------|
| `loading` | `isLoading`/`isFetching` | скелетон/спиннер |
| `success` | данные есть | контент |
| `empty` | данные пусты | заглушка + CTA |
| `error/{код}` | `isError` | сообщение по коду: 400 (валидация), 401 (переавторизация), 403 (нет доступа — скрыть/редирект), 404 (не найдено), 409 (конфликт), 500 (общая ошибка) |
| `no-access` | 403 / нет прав | секция скрыта или редирект |

## Маппинг API → UI

- Поля брать из сгенерированных **моделей** (`generated/models`), не из «сырых» ответов.
- Преобразования для отображения (формат дат, агрегаты) — в `model`-сегменте среза или `shared/lib`, не в JSX.

## Чего избегать

- Самописный `fetch`/`axios` в компонентах вместо сгенерированных хуков.
- Игнор состояний `error`/`empty`/`no-access` (показывать только happy path).
- Дублирование query-ключей и моделей вручную.
