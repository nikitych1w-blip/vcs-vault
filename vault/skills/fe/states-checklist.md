# FE · Состояния, доступ, чеклист разработки

Скилл-выжимка. Полная заметка — `../../knowledges/fe/development-checklist.md` (источник VCS-10805).

## Обязательные состояния экрана

| Состояние | Что делать |
|-----------|-----------|
| **loading** | `Spinner`/заглушка (`@sds-eng/base`), не пустой экран |
| **error** | понятное сообщение (без stack trace/raw error), кнопка «Повторить» где уместно; `PageError type="SERVER_ERROR"` |
| **empty** | `PageError type="EMPTY"`; различать «нет данных вообще» vs «нет совпадений по фильтру» |
| **content** | безопасно обрабатывать `undefined`/`null`; lazy-loading/виртуализация для больших списков |

## Права доступа (access)

- UI адаптируется под доступ пользователя (ручки `/access/repo`, `/access/project`).
- Права **всегда** проверяются на бэкенде — UI-ограничения не заменяют их.
- Отсутствие прав не должно ломать рендер; не должно быть «мерцающих» элементов при разных ролях.

## data-testid

На ключевые интерактивные элементы (кнопки/формы/таблицы/модалки), семантичные имена (`project-create-repo-button`), уникальные в компоненте, обязательно для элементов с условным рендером. (Стандарт — [[styleguide]].)

## Standalone / MFE

Микрофронтенд работает и в составе хоста (MFE), и в standalone — проверить standalone-сборку на стенде перед мержем.

## Ссылки Legacy ↔ React

- **`LinkRouter`** — переход внутри ReactApp (бесшовно).
- **`Link` (из sds)** — переход на LegacyUI (Go Templates): hard-reload (нужно для инициализации GiteaBridgeMFE).
- Избегать `full-page reload` при переходах внутри нового фронта.
